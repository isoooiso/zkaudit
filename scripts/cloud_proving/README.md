# Cloud proving workflow (RISC Zero / Groth16)

Run RISC Zero proving on a temporary **Ubuntu 24.04 x86_64** VPS instead of slow local WSL downloads. Your Windows machine only needs git and SSH; the server installs Rust, RISC Zero, and runs proofs.

**No secrets belong in this repo.** Do not commit `.env`, keys, or server passwords.

---

## A. Local preparation (Windows / WSL)

### 1. Stop local rzup downloads

If a previous `rzup install` is still running in WSL, cancel it so it does not compete for bandwidth:

```powershell
# In PowerShell â€” list WSL processes related to rzup
wsl bash -lc "ps aux | grep -E 'rzup|curl.*risc0' | grep -v grep"

# Kill stuck installs if needed (adjust PID from the list above)
wsl bash -lc "pkill -f 'rzup install' || true"
```

### 2. Check git status

From your project folder (`C:\Users\petre\Desktop\zkaudit`):

```powershell
cd C:\Users\petre\Desktop\zkaudit
git status
git log --oneline -3
```

Commit any workspace/host changes before pushing.

### 3. Check for secrets (do not push keys)

```powershell
cd C:\Users\petre\Desktop\zkaudit
git grep -i -E "secret|password|private.key|api_key|BEGIN RSA|BEGIN OPENSSH" -- . ":(exclude).gitignore" || echo "No obvious secrets in tracked files."
```

Also confirm `.env` and `*.pem` are listed in `.gitignore` and not staged:

```powershell
git status --ignored | findstr /i "env pem key"
```

### 4. Push to GitHub

```powershell
git remote -v
git push -u origin master
```

If the repo is private, use a GitHub personal access token or SSH key **on your machine only** â€” never commit tokens.

## B. Create a cloud server

| Setting | Recommendation |
|--------|----------------|
| OS | Ubuntu 24.04 LTS |
| Arch | x86_64 |
| Size | 8 vCPU / 16 GB RAM (Groth16 + Docker) |
| Region | EU (or closest to you) |
| Disk | 40+ GB |

Providers: Hetzner, DigitalOcean, AWS EC2, etc.

Connect:

```bash
ssh root@SERVER_IP
```

Replace `SERVER_IP` with the public IP from your provider. Use SSH keys; do not put keys in the repo.

## C. On the server (copy-paste)

Install git, clone, and run proving inside **tmux** so disconnects do not kill long builds.

```bash
apt update && apt install -y git
git clone https://github.com/isoooiso/zkaudit.git
cd zkaudit
chmod +x scripts/cloud_proving/*.sh
tmux new -s zkaudit
```

Inside the tmux session:

```bash
bash scripts/cloud_proving/bootstrap_ubuntu.sh
bash scripts/cloud_proving/run_dev_mode.sh
bash scripts/cloud_proving/run_real_groth16.sh
```

**Order matters:** bootstrap once per server; dev mode sanity-checks the guest; real Groth16 takes several minutes (Docker image pull on first run).

## E. Download outputs on Windows

1. Edit `scripts/cloud_proving/fetch_outputs.ps1`:
   - Set `$ServerIp = "YOUR.VPS.IP.HERE"`
   - Adjust `$ServerUser` / `$RemoteRepoPath` if you did not use root or `/root/zkaudit`

2. Run from PowerShell:

```powershell
cd C:\Users\petre\Desktop\zkaudit
powershell -ExecutionPolicy Bypass -File scripts\cloud_proving\fetch_outputs.ps1
```

Artifacts land in `C:\Users\petre\Desktop\zkaudit\cloud-artifacts\`.

Requires OpenSSH client (`scp`). Uses your normal SSH key/agent â€” no password in the script.

## G. Troubleshooting

### Git clone fails (private repo)

- Use SSH clone: `git clone git@github.com:isoooiso/zkaudit.git` (add deploy key on server)
- Or HTTPS with a token entered at prompt (never commit the token)

### `ERROR: No Cargo.toml at repository root`

The RISC Zero workspace is not in the repo yet. On a machine with `cargo risczero`:

```bash
cargo risczero new zkvm --guest-name audit_engine
# implement guest + host, commit, push, re-clone or git pull on server
```

### Cargo package is not named `host`

Scripts auto-detect the host package (`host/`, or any crate with `risc0-ethereum-contracts` / `methods`). If detection fails, the log prints workspace members â€” rename or add `host/` per RISC Zero template.

### `rzup: command not found`

```bash
export PATH="$HOME/.risc0/bin:$HOME/.cargo/bin:$PATH"
source ~/.cargo/env
bash scripts/cloud_proving/bootstrap_ubuntu.sh
```

### tmux session lost / SSH dropped

```bash
tmux attach -t zkaudit
# or check if cargo still running:
ps aux | grep cargo
```

### Artifacts not found after Groth16

- Check `logs/cp3-real-groth16.log` for errors (Docker, version mismatch).
- Host may write `proof.txt` / `journal.hex` at repo root; `run_real_groth16.sh` copies them into `artifacts/`.
- If the host only prints to stdout, update `host/src/main.rs` to write:
  - `artifacts/proof.txt`
  - `artifacts/journal.hex`

### Docker required for Groth16

Install Docker on Ubuntu:

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker "$USER"
# log out and back in, then:
docker ps
```

Re-run `bash scripts/cloud_proving/run_real_groth16.sh`.

### RISC Zero version mismatch with Stellar verifier

Pin `risc0-zkvm` and `risc0-ethereum-contracts` in `host/Cargo.toml` to the version in `vendor/stellar-risc0-verifier` (Checkpoint 1). Regenerate proof after pinning.
